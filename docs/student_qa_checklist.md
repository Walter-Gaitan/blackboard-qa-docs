# Student QA Checklist

**Goal**: Verify the course is student-ready by checking navigation, content visibility, and link validity.

## 1. Course Info & Orientation
- [ ] **Course Code**: Verify it matches the header and banner.
- [ ] **Banner**: Ensure image matches the course title/code.
- [ ] **Start Here / Orientation Module**:
    - [ ] Must be visible to students.
    - [ ] Contains "Student Resources" (or Program Resources).
    - [ ] Contains clear instructions for getting started.
- [ ] **Syllabus**:
    - [ ] Link is valid and opens the correct document.
    - [ ] Header info (Code/Name) is correct.
    - [ ] No "Track Changes" or visible comments.
    - [ ] Format is clean and professional.
- [ ] **Acknowledgement**: "Course Orientation Acknowledgement" test exists.

## 2. Modules & Navigation
- [ ] **Visibility**: All learning modules are set to "Visible to students".
- [ ] **Release Conditions**: Verify any conditional releases are set up correctly.
- [ ] **Tabs**:
    - [ ] **Discussions**: Topics follow a logical order.
    - [ ] **Gradebook**: Columns are organized logically.
    - [ ] **Groups**: Self-enrollment groups are set up (if applicable).

## 3. Content Checks (Assignments, Discussions, Tests, Journals)
*For every item type found in the course:*
- [ ] **Banner**: Matches the activity type.
- [ ] **Instructions**: Clear guidelines provided.
- [ ] **Rubric**: Attached, visible to students, and matches SharePoint files.
- [ ] **Links/Attachments**:
    - [ ] Open in a new tab (`target="_blank"`).
    - [ ] No "raw" URLs (e.g., `https://...`) visible in text.
    - [ ] All links are valid (no 404s).

## 4. Final Wrap-up
- [ ] **Tracking List**: Update status to "SQA findings".
- [ ] **Time Log**: Record time spent on QA.
